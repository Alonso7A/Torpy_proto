La seguridad en GitHub se realiza mediante claves SSH(ssh-keygen), es un método robusto y altamente seguro, basado en criptografía de clave pública. Permite autenticarse sin contraseñas, usando una pareja de llaves (privada en local, pública en GitHub) para cifrar la conexión, evitando robos de credenciales. Los siguientes comandos permiten una clave segura:

# Comando 1: `ssh-keygen -t rsa -b 4096 -C ""`

Este comando crea un par de llaves SSH (pública y privada)

**`ssh-keygen -t rsa -b 4096 -C ""`**

- `-t rsa`: Especifica el tipo de algoritmo (RSA)
    
- `-b 4096`: Crea una llave de 4096 bits (muy segura)
    
- `-C ""`: Agrega un comentario, deberías poner tu email, ej: `-C "tu@email.com"`

Los archivos generados son dos:

- `~/.ssh/id_rsa` → **LLAVE PRIVADA** (NUNCA compartir)
    
- `~/.ssh/id_rsa.pub` → **LLAVE PÚBLICA** (la que das a GitHub)

# Comando 2: `eval "$(ssh-agent -s)"`

Este comando inicia el "agente SSH" en segundo plano, también permite ya no escribir su contraseña cada vez que se usa Git con GitHub y guarda la llave en la memoria.

**`eval "$(ssh-agent -s)"`**

- `ssh-agent`: Programa que guarda tus llaves en memoria
    
- `eval`: Configura las variables de entorno necesarias
    
- `-s`: Muestra los comandos necesarios para configurar el agente

# Comando 3:`ssh-add ~/.ssh/id_rsa`

Este comando agrega la llave privada al agente SSH, otorgar la llave privada y se puede usar cuando GitHub pida autenticación.

**`ssh-add ~/.ssh/id_rsa`**

- `ssh-add`: Añade identidades al agente
    
- `~/.ssh/id_rsa`: Ruta de tu llave privada

# Comando 4: `sudo apt install xclip` o `cat ~/.ssh/id_rsa.pub`

Estos comando se usan para copiar la clave de seguridad pública, es más recomendable el segundo comando.

# SSH en GitHub

Para agregar la clave de seguridad anterior, sigue los siguientes pasos:

- **PASO 1:** Entrar al menú de GitHub y seleccionar la opción *SSH and GPS keys*.
	
- **PASO 2:** En Title, escribir el nombre del dispositivo que creó la llave, Ejm: Laptop. En Key-type mantenerlo en "Authentication Key" y en el recuadro de Key pegar el código copiado anteriormente.
	
- **PASO 3:** Dar clic en *Add SSH key* y en SSH keys, aparecerá la clave de autenticación creada y con el título otorgado.
