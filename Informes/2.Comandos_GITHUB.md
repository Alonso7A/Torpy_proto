Existen muchos comandos que utiliza Git y que se deben emplear para trabajar correctamente con nuestro repositorio en GitHub.

# Configuración inicial

| Comando                                  | Función                         |
| ---------------------------------------- | ------------------------------- |
| `git config --global user.name "nombre"` | Configura tu nombre de usuario  |
| `git config --global user.email "email"` | Configura tu correo electrónico |
| `git config --list`                      | Muestra la configuración actual |

# Comandos Básicos

|Comando|Función|
|---|---|
|`git init`|Inicializa un repositorio Git en el directorio actual|
|`git clone [url]`|Clona un repositorio remoto a tu máquina local|
|`git status`|Muestra el estado de los archivos (modificados, nuevos, etc.)|
|`git add [archivo]`|Agrega archivos al área de staging|
|`git add .`|Agrega todos los archivos modificados al staging|
|`git commit -m "mensaje"`|Crea un commit con los archivos en staging|
|`git log`|Muestra el historial de commits|

# Sincronización con GitHub

|Comando|Función|
|---|---|
|`git remote add origin [url]`|Conecta el repositorio local con GitHub|
|`git remote -v`|Muestra los repositorios remotos configurados|
|`git push origin [rama]`|Sube los cambios a GitHub|
|`git pull origin [rama]`|Descarga y fusiona cambios desde GitHub|
|`git fetch`|Descarga cambios sin fusionarlos|

# Ramas (Branches)

|Comando|Función|
|---|---|
|`git branch`|Lista las ramas locales|
|`git branch [nombre]`|Crea una nueva rama|
|`git checkout [rama]`|Cambia a otra rama|
|`git checkout -b [rama]`|Crea y cambia a una nueva rama|
|`git merge [rama]`|Fusiona otra rama con la actual|
|`git branch -d [rama]`|Elimina una rama|

# Comandos Avanzados

|Comando|Función|
|---|---|
|`git diff`|Muestra las diferencias en archivos modificados|
|`git reset [archivo]`|Quita archivos del área de staging|
|`git stash`|Guarda cambios temporalmente|
|`git stash pop`|Recupera los cambios guardados|
|`git tag [nombre]`|Crea una etiqueta para marcar versiones|

# Comandos Útiles

| Comando                     | Función                                  |
| --------------------------- | ---------------------------------------- |
| `git rm [archivo]`          | Elimina archivos del repositorio         |
| `git mv [origen] [destino]` | Renombra o mueve archivos                |
| `git show [commit]`         | Muestra detalles de un commit específico |
| `git blame [archivo]`       | Muestra quién modificó cada línea        |

Es posible usar rm y mv, pero con git rm y git mv, automáticamente se encuentra en staging y en los comandos de consola se necesita otro git add para confirmar.
