# Generación de Token para GIT PUSH

También llamado Personal Access Token - PAT, es la creación de una contraseña alternativa y segura, exigida por plataformas como GITHUB para autenticar operaciones en la línea de comandos, reemplazando la contraseña de tu cuenta para mejorar la seguridad al subir cambios en Git Hub.

## ¿Qué es un Token?

Es un código único que funciona como contraseña temporal o permanente, con permisos específicos. GitHub ya **no acepta la contraseña real** de la cuenta en operaciones de terminal.

## Procedimiento para crear este Token

Este Token creado, servirá al momento de ejecutar un Git push, cuando requiera tu `password`.

```
Username for 'https://github.com': tu-usuario-github
Password for 'https://tu-usuario@github.com': Aquí se coloca el Token
```

| Paso | Acción                                                                               | Captura conceptual |
| ---- | ------------------------------------------------------------------------------------ | ------------------ |
| 1    | Iniciar sesión en **[github.com](https://github.com/)**                              | -                  |
| 2    | Hacer clic en **foto de perfil** (esquina superior derecha)                          | 👤                 |
| 3    | Seleccionar **Settings**                                                             | ⚙️                 |
| 4    | En el menú izquierdo, desplazarse hasta abajo y hacer clic en **Developer settings** | 🔧                 |
| 5    | Hacer clic en **Personal access tokens** → **Tokens (classic)**                      | 🔑                 |
| 6    | Hacer clic en **Generate new token** → **Generate new token (classic)**              | ➕                  |
Después de este último paso, se puede configurar el Token y cómo utilizarlo.

- *NOTE:* En este apartado colocar un nombre para utilizar el token, ejemplo `token git push [laptop]`
	
- *EXPIRATION:* Colocar el tiempo de vida del token, generalmente se coloca `60 o 90 days` 
	
- *ESCOPE:* Marcar `repo` para permitir control total sobre los repositorios

# Creación de carpetas en el repositorio

Para crear nuevas carpetas dentro del repositorio con varios archivos, es necesario seguir estos pasos:

|Paso|Acción|
|---|---|
|1|Entrar al repositorio en **[github.com](https://github.com/)**|
|2|Hacer clic en **Add file** (botón verde)|
|3|Seleccionar **Create new file**|
|4|En el campo de nombre, escribir: `nombre-carpeta/` (el slash `/` es obligatorio)|
Luego de nombrar a la carpeta, se mostrará en la web para poder editar; sin embargo, en Visual Studio Code no aparecerá.

## Edición de la nueva carpeta

Para visualizar la carpeta y editar para agregar archivos a dicha carpeta, se deben ejecutar los siguientes comandos.

- Colocar `git branch` en el terminal de Visual Studio Code y se mostrará el nombre en `*` de la rama principal.
	
-  En la misma terminal ejecutar esta línea `git pull origin "master"` en este caso se llama master la branch principal.
