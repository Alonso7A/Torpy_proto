# ¿Qué es GitHub?

GitHub es una *plataforma en línea* para programadores, donde se puede almacenar, compartir y colaborar en proyectos de software, está basado en *Git*, sistema de control de versiones.

Es capaz de compartir códigos a la nube para tenerlos de respaldo y acceder a ellos desde cualquier computadora, guarda el historial completo de los cambios del proyecto y es posible volver a una versión anterior si algo sale mal, asimismo es posible trabajar con varias personas al mismo tiempo, se ha popularizado tanto GitHub que los repositorios y proyectos usados sirven de portafolio profesional, ya que son *Open Source*, código modificable.

- **Git** sería el cuaderno donde anotan todos los cambios, quién los hizo y cuándo.
    
- **GitHub** sería una biblioteca en internet donde guardan ese cuaderno, y que además tiene mesas para reunirse, ventanas para que otros vean lo que escriben, y estantes para organizar los capítulos.

# Programas necesarios

Primero es necesario preparar el ordenador y la cuenta de Git Hub para crear el primer repositorio:

- *Crear una cuenta en GitHub:* Debes ir al link de la página oficial de GitHub https://github.com/, en caso no tener aún cuenta, seleccionar la opción **"create account"** y rellenar el usuario, el correo y la contraseña, después ya puedes colocar tus datos anteriores y presionar **"Sign in"**.
- *Instalar un editor de código:* Es posible usar cualquier editor de código de su preferencia; sin embargo, es recomendable instalar **Visual Studio Code**, ya que tiene una terminal integrada y soporte nativo para Git, lo que facilita mucho el proceso, aquí el link de instalación https://code.visualstudio.com/. 
- *Instalar Git en el ordenador:* Git es el sistema de control de versiones que se usa para sincronizar los cambios de los proyectos y permite a múltiples desarrolladores trabajar simultáneamente sin sobrescribir un trabajo ajeno. Ve a la página oficial de Git https://git-scm.com/downloads, seleccionar el sistema operativo y selecciona la última versión de Git en **"Click here to download"** en caso ser Windows, otros sistemas operativos requieren comandos para instalarlos.

# Creación de Repositorio y Configuración

Una vez iniciada la sesión en GitHub e instalado los demás softwares se deben seguir los siguientes pasos:

- **PASO 1:** En la pestaña de GitHub, ir al menú con el ícono "triple raya" en la izquierda superior, seleccionar en *Repositories*.

- **PASO 2:** Elegir *New repository*, seleccione un nombre y una descripción acerca del proyecto a realizar, *dejar en público el repositorio* (recomendado) porque al colocarlo en privado puede que no reconozca el repositorio y no mostrar en donde se ubican los errores.

- **PASO 3:** Dar clic en *Create repository* y te llevará a la pestaña de tu repositorio, aparecerán instrucciones para conectar con tus archivos locales.

- **PASO 4:** Abrir una carpeta nueva en *Visual Studio Code*, abrir un nuevo *terminal* y escribir el comando`git init`, en caso que todo funcione deberá aparecer `Initialized empty Git repository in ...`

- **PASO 5:** Escribir el sexto comando que aparecen en las instrucciones del GitHub`git remote add origin https://github.com/""/""` para vincular el repositorio local con el remoto, al ejecutarlo no aparecerá nada en el terminal (es normal). 

- **PASO 6:** Escribir `git status` aparecerán los archivos que Git no reconoce si hay cambios, para que los reconozcan utilizar `git add "nombre_del_archivo"` o para reconocer todos los archivos existentes `git add .` y en el terminal tampoco aparecerá nada; sin embargo, si volvemos a escribir `git status` debe aparecer el o los archivos de color *verde*.

> [!ESPECIAL] Otro comando
> Es posible utilizar `git commit -m "descripcion"` para crear puntos de guardado, por si hay un error y queremos regresar a un commit anterior.

- **PASO 7:** Escribir `git commit -m "descripcion"` y escribir de nuevo `git status`, si en este último aparece "On branch master" y "nothing to commit", se escoge la *branch* donde se colocarán los archivos con `git push --set-upstream origin master` o copiar ese mismo comando cuando ejecutes y te salga un error al poner `git push`.

- **PASO 8:** Después del `git push --set-upstream origin master`, nos aparecerá que se subió el cambio a GitHub.

- **PASO 9:** Verificar en la página de GitHub donde se ubica tu repositorio, ya no se verán las instrucciones, sino los archivos. Recordar el orden al crear un nuevo archivo. `git status`*(aparecerá en rojo)* → `git add .` → `git status`*(aparecerá en verde)* → `git commit -m "xxxxx"` → `git push`.

En algunos casos se pedirá configurar Git, cuando se instala por primera vez, es por ello que se debe rellenar `git config --global user.name "tu nombre"` y `git config --global user.email "tucorreo@mail.com` .
